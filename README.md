From a sketch to a full End-to-End Data Analysis Project

Recently, I decided to go beyond just learning tools and actually apply the full Data Life Cycle through a "Store Management System" project. It was a journey from scratch:

 The Design Phase (Database Design): It all started with analyzing business requirements and building the ERD and Mapping. My goal was to ensure solid relationships between tables (Customers, Orders, Products, Employees) while maintaining proper Normalization to avoid redundancy.

 The Implementation Phase (SQL Server): I turned that design into reality using SQL. I built the Schema from the ground up and performed Data Insertion with simulated real-world data. I didn't just use SQL for storage; I used it for Data Cleaning and created Views to ensure the data was analysis-ready.

 The Visualization Phase (Power BI): Connecting the database to Power BI is where the real magic happened:

Designed a robust Data Model (Star Schema) for optimal performance.

Leveraged DAX to calculate critical KPIs (like Average Order Value per Customer and Total Revenue).

Focused the dashboard on answering key business questions: "Which category drives the most profit?", "What is the preferred payment method?", and "Employee performance analysis based on sales."

The goal was to prove to myself that a great analyst doesn't just create charts—they understand where the data comes from and how it’s engineered from day one.

I’d love to hear your feedback on the design and the analysis!
